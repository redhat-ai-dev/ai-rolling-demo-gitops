# RHDH AI Checklist

- [ ] Refer to the checklist items on how to create Epic(s) and assign Components [RHDH Feature Exploration Process](https://docs.google.com/document/d/1VOPv4ahpBhEFpLOWuJICxRxCuO1m09pyiqDM5t64F8I/edit?tab=t.0#heading=h.wijln92hiyvx)  
- [ ] For RHDH AI Feature sizing, identify the Epic(s) to be created for the RHDH AI team and other Dependencies like Docs, UI, etc. on the Refinement Notes (Example [RHDH AI 1.10 Early Feature Refinement notes](https://docs.google.com/document/d/1_JaLsICKpkLuca0cctcK94umLAF4RTFw5kTdy6YEQh8/edit?tab=t.0#heading=h.mrzl1resn3u4)[RHDH AI 2.1 Early Feature Refinement notes](https://docs.google.com/document/u/0/d/1GMUZ1lhXEn0yeZOrYz77-aFIlt9Rw90Aj_dSzSWECLE/edit))  
      - [ ] List the Epics that are going to be worked on by the RHDH AI team in the early feature refinement doc with the following info:  
            - [ ] Name of epic  
            - [ ] Sizing in Person Days (PD), numbers only and not a range  
            - [ ] MVP/Stretch  
- [ ] **For QE related dependencies**, open an Epic under the Feature with \[QE\] in the title and assign   
      - [ ] Component:  Quality  
      - [ ] Team: RHDH AI  
- [ ] **For Doc related dependencies**, follow the automation process described in [RHDH Feature Exploration Process](https://docs.google.com/document/d/1VOPv4ahpBhEFpLOWuJICxRxCuO1m09pyiqDM5t64F8I/edit?tab=t.0#heading=h.wijln92hiyvx)  
- [ ] **For UI related dependencies**, open an Epic under the Feature and assign  
      - [ ] Component: Lightspeed (for example)  
      - [ ] Team: RHDH Frontend Plugins & UI  
- [ ] **For UX related dependencies**, open an Epic under the Feature on the [DTUX project](https://issues.redhat.com/projects/DTUX) and assign  
      - [ ] Component: RHDH  
      - [ ] Labels: developer-lightspeed
